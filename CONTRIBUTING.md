# Contributing

Grazie per voler dare una mano.

Questo progetto e' un laboratorio di reverse engineering prima ancora che una libreria rifinita. I contributi piu' utili sono quelli che migliorano una di queste cose:

- riproducibilita' dei test;
- chiarezza della documentazione;
- pulizia degli script esistenti;
- nuove catture o verifiche sul protocollo;
- piccoli strumenti che rendano il workflow piu' semplice da usare.

## Prima di aprire una PR

- descrivi cosa hai provato davvero sulla stampante;
- indica sistema operativo, script usato e comando eseguito;
- se tocchi il protocollo, includi payload, risposta osservata ed effetto fisico;
- se puoi, collega una cattura o un file in `captures/`;
- se la cattura deve finire nel repo pubblico, pubblica una versione sanitizzata in `captures/public/`.

## Stile del progetto

- meglio cambi piccoli e leggibili che grandi riscritture;
- preferiamo note verificabili e comandi ripetibili;
- quando un comportamento non e' certo, dichiararlo esplicitamente e' meglio che presentarlo come fatto.

## Issue

Una issue fatta bene dovrebbe dire almeno:

- cosa volevi ottenere;
- quale script hai usato;
- quale comando hai lanciato;
- cosa e' successo davvero;
- se la stampante ha mosso carta, stampato nero, risposto via notify o non ha fatto nulla.

## Aspettative

Il maintainership qui e' volutamente leggero. Il codice e' nato in modo pragmatico e non tutte le parti hanno dietro una teoria completa o una spiegazione elegante. Va bene cosi': se vuoi aiutare a consolidarlo, documentarlo meglio o renderlo piu' robusto, il contributo e' benvenuto.
